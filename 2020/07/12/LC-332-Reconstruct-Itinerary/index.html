LC#332 Reconstruct Itinerary

Posted on 2020-07-12

Problem

Given a list of airline tickets represented by pairs of departure and arrival airports [from, to], reconstruct the itinerary in order. All of the tickets belong to a man who departs from JFK. Thus, the itinerary must begin with JFK.

Note:

  1. If there are multiple valid itineraries, you should return the itinerary that has the smallest lexical order when read as a single string.
  2. All airports are represented by three capital letters (IATA code).
  3. You may assume all tickets form at least one valid itinerary.
  4. One must use all the tickets once and only once.

Analysis

Basically this is a graph problem, so we can use DFS to traverse the whole graph and find the valid itinarary.

First of all, we need to build the graph, which is to store the <origin, destList> pair in a map. Then in order to meet the lexical order requirement, we need to sort the destinations for each origin. Also we need a variable visitedMap to track if this destination is visited, so we initialize the visitedMap all false in this step.

Then it comes to backtrack part. Before that, we need a variable route to track the path, and a variable flights to count the number of flights and check if we visited all flights. So the end condition is just route.size() == flights + 1. Then for the recursive part, we try all destinations under current origin, until find a valid flight.

Complexity Analysis

  • Time Complexity: O(Ed), where E is the number of total flights and d is the maximum number of flights from an airport.
  • Space Complexity: O(V + E) where V is the number of airports and E is the number of flights.
