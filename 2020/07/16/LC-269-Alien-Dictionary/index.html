LC#269 Alien Dictionary

Posted on 2020-07-16

Problem

There is a new alien language which uses the latin alphabet. However, the order among letters are unknown to you. You receive a list of non-empty words from the dictionary, where words are sorted lexicographically by the rules of this new language. Derive the order of letters in this language.

Example:

Note:

  • You may assume all letters are in lowercase.
  • If the order is invalid, return an empty string.
  • There may be multiple valid order of letters, return any one of them is fine.

Analysis

In general, there are three steps to solve this problem:

  1. Extracting dependency rules from the input.
  2. Building a graph with the dependency rules.
  3. Topologically sorting the graph.

The most important point is the topological sort, we can use the stratege that showed below. Which is to add those nodes that have no going into arrow every time.

So that we can get the following sorting result:

So how to implement this strategy? We can use two maps, one stores the list of adjacent nodes of the key node (just like the common way to build a graph), one stores the number of its pre-nodes (the number of going-into arrows). And with BFS, each time we just poll out one char from the queue, append it to our result (since all the char in the queue just have no going-into arrow), update pre-nodes-map value of its adjacent nodes (decrese by 1), then add the char whose pre-nodes-map value equals to 0 to the queue again.

Notice: It is not a good way to extract dependency rules just through the vertically corresponding position (like getting the rules based on all first letters, then based on all second letters). The right way to get the rules is from the first different letter of two adjacent words, which means we get the sequence of every two letters firstly.

Tips:

  • Use .equals() for Integer, dont forget!
  • There are two invalid cases:
    1. When the latter word shorter than the former one, and they have the same prefix.
    2. When our final result doesnt contain all characters.
  • Notice the following process:
    • BFS initialization: add all 0-going-into-arrow nodes in the queue, no need to update preMap here, no need to append these chars to the result sb.
    • Every time poll one element from the queue, add this char to the result sb, and for all its adjacents, update their preMaps, and add the 0-going-into-arrow node into the queue.

Complexity Analysis

N: the total number of strings in the input list.
C: the total length of all the words in the input list, added together.
U: the total number of unique letters in the alien alphabet. (26 in this problem)

  • Time Complexity: O(C) (worst case, check every chars)
  • Space Comlexity: O(1) or O(U + min(U2, N)) (Mostly for the adjacency list)
