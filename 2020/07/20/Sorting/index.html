Sorting

Posted on 2020-07-20

Bubble Sort

Compare two elements of unsorted portion one by one, if the former one is bigger than the latter one, swap them.

Compare n-1 times in total.

Unsorted portion is at the last of the array, so we only need to compare in the range [0, n-i-1).

If there is no swap in current loop, we can terminate in advance.

Complexity Analysis

  • Time Complexity: O(N2)
  • Space Complexity: O(1)

Selection Sort

Repeatedly select the smallest remaining item and move it to the end of sorted portion.

Complexity Analysis

  • Time Complexity: O(N2)
  • Space Complexity: O(1)
  • Unstable

Heap Sort

Instead of rescanning entire array for minimum item, we can maintain a heap to find it fast.

In-place Bottom-up Heapification. Efficient heap construction by sinking nodes in reverse level order.

Reapt N times:

  1. O(log N) Delete max item from the max-heap.
  2. O(1) Put max item at the end the array.

Complexity Analysis

  • Time Complexity: O(N log N)
  • Space Complexity: O(1)
  • Unstable

Merge Sort

Merge Sort Steps:

  1. If array is of size 1, return.
  2. Merge sort the left half.
  3. Merge sort the right half.
  4. Merge the two sorted halves.

Complexity Analysis

  • Time Complexity: O(N log N)
  • Space Complexity: O(N)
  • Stable

Insertion Sort

Build a sorted subarray (like selection sort) by using left-neighbor swaps for stability.

Scan from left to right …

  1. If an item is out of order with respect to its left-neighbor, swap left.
  2. Keep on swapping left until the item is in order with respect to its left-neighbor.

Complexity Analysis

  • Time Complexity: O(N2)
  • Space Complexity: O(1)
  • Stable

Quick Sort

Naive Quick Sort:

  1. Partition around a pivot item, e.g. leftmost item.
  2. Quicksort left side, all keys < pivot.
  3. Quicksort right side, all keys ≥ pivot.

Complexity Analysis

  • Time Complexity: Normal case O(N log N) Worst case O(N2)
  • Space Complexity: O(1)
  • Stable
