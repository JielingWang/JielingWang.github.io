Sorting

Posted on 2020-07-20

Bubble Sort

Compare two elements of unsorted portion one by one, if the former one is bigger than the latter one, swap them.

Compare n-1 times in total.

Unsorted portion is at the last of the array, so we only need to compare in the range [0, n-i-1).

If there is no swap in current loop, we can terminate in advance.

Complexity Analysis

  • Time Complexity: O(N2)
  • Space Complexity: O(1)

Selection Sort

Repeatedly select the smallest remaining item and move it to the end of sorted portion.

Complexity Analysis

  • Time Complexity: O(N2)
  • Space Complexity: O(1)
  • Unstable

Heap Sort

Instead of rescanning entire array for minimum item, we can maintain a heap to find it fast.

In-place Bottom-up Heapification. Efficient heap construction by sinking nodes in reverse level order.

Reapt N times:

  1. O(log N) Delete max item from the max-heap.
  2. O(1) Put max item at the end the array.

Complexity Analysis

  • Time Complexity: O(N log N)
  • Space Complexity: O(1)
  • Unstable

Merge Sort

Merge Sort Steps:

  1. If array is of size 1, return.
  2. Merge sort the left half.
  3. Merge sort the right half.
  4. Merge the two sorted halves.

Complexity Analysis

  • Time Complexity: O(N log N)
  • Space Complexity: O(N)
  • Stable

Insertion Sort

Build a sorted subarray (like selection sort) by using left-neighbor swaps for stability.

Scan from left to right …

  1. If an item is out of order with respect to its left-neighbor, swap left.
  2. Keep on swapping left until the item is in order with respect to its left-neighbor.

Complexity Analysis

  • Time Complexity: O(N2)
  • Space Complexity: O(1)
  • Stable

Quick Sort

Naive Quick Sort:

  1. Partition around a pivot item, e.g. leftmost item.
  2. Quicksort left side, all keys < pivot.
  3. Quicksort right side, all keys ≥ pivot.

Complexity Analysis

  • Time Complexity: Normal case O(N log N) Worst case O(N2)
  • Space Complexity: O(1)
  • Stable

Problem

On a 2x3 board, there are 5 tiles represented by the integers 1 through 5, and an empty square represented by 0. A move consists of choosing 0 and a 4-directionally adjacent number and swapping it. The state of the board is solved if and only if the board is [[1,2,3],[4,5,0]].

Given a puzzle board, return the least number of moves required so that the state of the board is solved. If it is impossible for the state of the board to be solved, return -1.

Analysis

When it comes to the least step, the burning approach should be BFS. To use BFS to solve this problem, there are several points need to be noticed.

  • How to represent the state of current board?

We can use a String with 6-length, the character on each position represents the number on the corresponding position on the board. So the target should be "123450".

  • How to find the adjacency node?

Since there are only 6 possible positions in this problem, so that we can store the adjacency positions of each position in an array ahead, and each time we just need to know where the 0 is to find its adjacency position.

Code

Complexity Analysis

  • Time Complexity: O((R * C)!), where R, C are the number of rows and columns of board. There are o((R * C)!) possible board states.
  • Space Complexity: O(R * C * (R * C)!)

Problem

There is a new alien language which uses the latin alphabet. However, the order among letters are unknown to you. You receive a list of non-empty words from the dictionary, where words are sorted lexicographically by the rules of this new language. Derive the order of letters in this language.

Example:

Note:

  • You may assume all letters are in lowercase.
  • If the order is invalid, return an empty string.
  • There may be multiple valid order of letters, return any one of them is fine.

Analysis

In general, there are three steps to solve this problem:

  1. Extracting dependency rules from the input.
  2. Building a graph with the dependency rules.
  3. Topologically sorting the graph.

The most important point is the topological sort, we can use the stratege that showed below. Which is to add those nodes that have no going into arrow every time.

So that we can get the following sorting result:

So how to implement this strategy? We can use two maps, one stores the list of adjacent nodes of the key node (just like the common way to build a graph), one stores the number of its pre-nodes (the number of going-into arrows). And with BFS, each time we just poll out one char from the queue, append it to our result (since all the char in the queue just have no going-into arrow), update pre-nodes-map value of its adjacent nodes (decrese by 1), then add the char whose pre-nodes-map value equals to 0 to the queue again.

Notice: It is not a good way to extract dependency rules just through the vertically corresponding position (like getting the rules based on all first letters, then based on all second letters). The right way to get the rules is from the first different letter of two adjacent words, which means we get the sequence of every two letters firstly.

Tips:

  • Use .equals() for Integer, dont forget!
  • There are two invalid cases:
    1. When the latter word shorter than the former one, and they have the same prefix.
    2. When our final result doesnt contain all characters.
  • Notice the following process:
    • BFS initialization: add all 0-going-into-arrow nodes in the queue, no need to update preMap here, no need to append these chars to the result sb.
    • Every time poll one element from the queue, add this char to the result sb, and for all its adjacents, update their preMaps, and add the 0-going-into-arrow node into the queue.

Code

Complexity Analysis

N: the total number of strings in the input list.
C: the total length of all the words in the input list, added together.
U: the total number of unique letters in the alien alphabet. (26 in this problem)

  • Time Complexity: O(C) (worst case, check every chars)
  • Space Comlexity: O(1) or O(U + min(U2, N)) (Mostly for the adjacency list)

For more detailed analysis, go to the link of reference.

LC#332 Reconstruct Itinerary

Posted on 2020-07-12

Problem

Given a list of airline tickets represented by pairs of departure and arrival airports [from, to], reconstruct the itinerary in order. All of the tickets belong to a man who departs from JFK. Thus, the itinerary must begin with JFK.

Note:

  1. If there are multiple valid itineraries, you should return the itinerary that has the smallest lexical order when read as a single string.
  2. All airports are represented by three capital letters (IATA code).
  3. You may assume all tickets form at least one valid itinerary.
  4. One must use all the tickets once and only once.

Example:

Analysis

Basically this is a graph problem, so we can use DFS to traverse the whole graph and find the valid itinarary.

First of all, we need to build the graph, which is to store the <origin, destList> pair in a map. Then in order to meet the lexical order requirement, we need to sort the destinations for each origin. Also we need a variable visitedMap to track if this destination is visited, so we initialize the visitedMap all false in this step.

Then it comes to backtrack part. Before that, we need a variable route to track the path, and a variable flights to count the number of flights and check if we visited all flights. So the end condition is just route.size() == flights + 1. Then for the recursive part, we try all destinations under current origin, until find a valid flight.

Code

Complexity Analysis

  • Time Complexity: O(Ed), where E is the number of total flights and d is the maximum number of flights from an airport.
  • Space Complexity: O(V + E) where V is the number of airports and E is the number of flights.

LC#393 UTF-8 Validation

Posted on 2020-07-05

Problem

A character in UTF8 can be from 1 to 4 bytes long, subjected to the following rules:

  1. For 1-byte character, the first bit is a 0, followed by its unicode code.
  2. For n-bytes character, the first n-bits are all one’s, the n+1 bit is 0, followed by n-1 bytes with most significant 2 bits being 10.

Here is how the UTF-8 encoding would work:

Given an array of integers representing the data, return whether it is a valid utf-8 encoding.
Note:
The input is an array of integers. Only the least significant 8 bits of each integer is used to store the data. This means each integer represents only 1 byte of data.

Analysis

This is not a problem which is difficult to implement, but it does have some challenges on understanding. So firstly we need to clarify the rules of a valid UTF-8 character:

  • A valid UTF-8 character can be 1 - 4 bytes long.
  • For a 1-byte character, the first bit is a 0, followed by its unicode.
  • For an n-bytes character, the first n-bits are all ones, the n+1 bit is 0, followed by n-1 bytes with most significant 2 bits being 10.
  • The input given would be an array of integers containing the data. We have to return if the data in the array represents a valid UTF-8 encoding. The important thing to note here is that the array doesnt contain data for just a single character. As can be seen from the first example, the array can contain data for multiple characters all of which can be valid UTF-8 characters and hence the charset represented by the array is valid.
  • Another important thing to note is that the integers in the array can be larger than 255 as well. The highest number that can be represented by 8 bits is 255. So, what do we do if an integer in the array is say, 476? According to the Note in the problem before the examples, we only have to consider the 8 least significant bits of each integer in the array.

So now we can solve this problem by bit manipulation. Some key points:

  • Use while to check how many 1s in a certain octet, so that determine how many bytes we need to process.
  • Remember that there might be some test cases that more than 4-bytes long, which are invalid.
  • Be sure to decrease byteNum after each process.
  • Use byteNum == 0 to ensure the final result.

Code

Complexity Analysis

  • Time Complexity: O(N)
  • Space Complexity: O(1)
