LC#149 Max Points on a Line

Posted on 2020-04-26

问题描述

Given n points on a 2D plane, find the maximum number of points that lie on the same straight line.

Example:

解决思路

首先需要考虑的一个问题是：怎么确定 a, b, c 三点在一条直线上？显然，只要确定直线 ab 和直线 ac 的斜率相等即可。同时斜率也可以作为这条直线的标识。所以这道题只需要计算出任意两点之间的斜率，用一个 HashMap 存储斜率和其对应点的个数，就可以找到处于同一斜率下的最多的点的个数。所以时间复杂度是 O(N2)

方法并不难，但是实现的时候有一些细节的地方需要考虑清楚：

  • 对于重复点的处理

在这道题目中，重复的点是需要算在内的。比如给出的例子中如果有两个[1,1]，那么最后输出的答案就是 5。因为每次是确定一个点作为基准点，计算基准点与剩下其他点之间的斜率，所以如果遇到与基准点重复的点就需要单独处理，因为斜率无法计算。

  • 斜率的表示

可以直接用一个数表示，但是就需要讨论水平和垂直直线的情况。这里直接用 x 和 y 最简分数表示，所以要求分子分母的最大公约数。下面的这个 GCD 函数可以作为模板记下来。这里无论 a 和 b 的顺序，计算出来的结果是一样的。

  • 如何避免重复计算

如果全局用一个 HashMap 实现，就会出现重复计算的问题。比如例子中四个在同一直线上的点，当以第一个点为基准点计算的时候，Map 中该斜率对应的点的个数就是 3，但是当再以第二点为基准点计算的时候，剩下的两个点依然会被算进去，Map 中该斜率对应的点的个数就变成了 5，这是不合理的。

解决思路有两个：

  1. 每次换基准点之前对 Map 进行清除，用一个 globalMax 变量来记录同斜率最大点数，在每次针对该基准点的所有操作完成后更新。
  2. HashMap 里不存相同斜率的点的个数，改存一个包含了所有点的 List，如果发现后来的点已经在 List 里了就可以不加。但是这样就需要重新考虑重复点的问题，如何判断一个点是之前已经加过所以不需要重复加到 List 里，还是这个点本身就是重复点所以需要加进去呢？显然第一条思路更好。

Lambda 表达式是 Java SE8 才加入的新变化。在这之前，因为 Java 是面向对象的，因此通常情况下代码段必须传递到某一个对象上，由对象调用才能执行。但是 Lambda 表达式可以直接传递代码段，在之后执行一次或多次。
